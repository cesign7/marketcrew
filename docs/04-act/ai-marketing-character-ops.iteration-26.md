# Iteration 26 - 브랜드 우선 메뉴/안건 구조 재정렬

## 완료한 목표

스티커씨와 커피프린트를 서로 비교하는 운영 구조를 제거하고, 상단 보기 기준과 데이터 연동 화면을 `브랜드 먼저, 판매채널은 하위` 원칙으로 재정렬했다.

## 비교 판단

| 참고 구조 | 관찰 | MarketCrew 반영 |
|---|---|---|
| Shopify 관리자 | 스토어를 기준으로 두고 판매채널을 사이드바/설정 안에서 관리한다. | 스티커씨/커피프린트를 먼저 선택하고, 스마트스토어/자체몰은 브랜드 안의 판매채널로 본다. |
| Google Ads 관리자 계정 | 매니저 계정 아래 고객 계정을 탐색하고, 현재 보고 있는 계정을 명확히 표시한다. | 광고 근거는 독립 채널이 아니라 선택된 브랜드의 판단 근거로 붙인다. |
| Cafe24 멀티쇼핑몰 | 한 계정 안에서 여러 쇼핑몰을 만들지만 개별 쇼핑몰의 상품/주문/고객/디자인 현황을 확인한다. | 커피프린트 스마트스토어 추가 예정은 커피프린트 브랜드 안의 판매채널 확장으로 다룬다. |

## 반영 내역

| 영역 | 변경 |
|---|---|
| 안건 생성 | `channel-balance` 교차 브랜드 안건 생성을 제거했다. 스마트스토어 안건은 프로, 커피프린트 자체몰 CRM 안건은 리피가 각각 상신한다. |
| AI 판독 근거 | `commerce_cross_channel` 통합 판독 근거를 제거했다. AI는 스마트스토어와 쇼핑몰 요약을 브랜드별로 따로 읽는다. |
| 메뉴/필터 | 상단 기준 라벨을 `브랜드`로 바꾸고 `커피`를 `커피프린트`로 풀어 썼다. |
| 데이터 화면 | 전체 근거는 한 화면에 보여주되, 설명 문구는 브랜드별 분리 판단을 명시한다. |
| 근거 확장 계획 | `스티커씨 스마트스토어 데이터솔루션`과 `커피프린트 스마트스토어 추가 준비`로 나눴다. |
| 문서 | AGENTS, plan, design에 브랜드 우선 원칙을 반영했다. |

## 테스트 기준

- 스티커씨와 커피프린트 매출을 하나의 균형 안건으로 묶지 않는다.
- 통합 AI 판독 키 `commerce_cross_channel`이 다시 나오면 테스트가 실패한다.
- 데이터 화면의 상단 보기 기준은 `브랜드`로 보인다.
- 커피프린트 스마트스토어 추가 예정은 준비 항목으로만 보이며 스티커씨와 직접 비교하지 않는다.
