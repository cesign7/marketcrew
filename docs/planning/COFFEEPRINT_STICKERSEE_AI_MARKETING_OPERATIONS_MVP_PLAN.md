# 커피프린트/스티커씨 AI 마케팅 운영실 MVP 기획서

작성일: 2026-05-20  
대상 브랜드: 커피프린트, 스티커씨  
1차 연결 우선순위: 네이버 검색광고 API  
목표: 마진 기반으로 네이버 광고, 키워드, 상품명, 광고문안, 시즌 캠페인 업무를 자동화하고 승인 흐름으로 안전하게 운영한다.

---

## 1. 기획 방향

기존 기획은 범용 “AI 마케팅 오피스”에 가까웠다. 이번 MVP는 범용 앱이 아니라 **커피프린트와 스티커씨 전용 AI 마케팅 운영실**로 좁힌다.

핵심 목표는 단순 추천이 아니다. 네이버 검색광고, 스마트스토어, 커피프린트 영카트, 원가 계산식을 연결해 **실제 이익이 남는 키워드와 상품을 찾고, 저위험 작업은 자동 실행하며, 위험하거나 큰 변경은 승인 요청**하는 것이다.

1차 MVP의 중심은 다음이다.

```txt
네이버 검색광고 데이터 수집
→ 과거 키워드 성과 분석
→ 핵심 키워드 후보 추출
→ 1위 방어 / 2~3위 유지 / 하향 / 제외 / 테스트 룰 생성
→ 저위험 입찰 조정 자동 실행
→ 승인 필요한 변경은 승인실로 전달
→ 실행 결과를 저장하고 룰을 개선
```

장기적으로는 블로그 SEO, CRM, 카카오 메시지, 메타/인스타, 구글 광고, 유튜브 Shorts까지 확장한다. 다만 범용 SaaS 전환은 내년 이후 목표로 두고, 먼저 내부 운영에서 실제 성과를 만든다.

## 3. 제품 컨셉

서비스 이름 가안: **AI 마케팅 운영실**

AI 에이전트들이 각자 캐릭터처럼 업무를 맡고, 사용자는 대표/관리자 입장에서 보고를 받고 승인한다.

예시:

```txt
순위 방어 AI
"기업 초대장 키워드가 목표 1위권에서 벗어났습니다.
5% 입찰 상향은 자동 허용 범위라 처리했습니다."

입찰 최적화 AI
"감사장 제작 키워드는 1위보다 2~3위권에서 공헌이익이 높습니다.
목표 순위를 2~3위 유지로 변경하는 것을 추천합니다."

상품명 SEO AI
"스티커씨 상품명 앞쪽에 '어린이집 생일 답례품'을 배치하는 안을 만들었습니다.
실제 변경은 승인 필요입니다."

총괄 매니저 AI
"오늘 자동 처리 4건, 승인 필요 7건, 위험 감지 2건입니다."
```

UI는 업무용 대시보드이지만 캐릭터, 말풍선, 업무카드, 퀘스트, 승인실을 활용해 게임적인 느낌을 준다. 단, 실제 업무 효율이 우선이다.

## 4. 핵심 목표

### 비즈니스 목표

- 광고비 낭비 감소
- 마진이 남는 키워드와 상품 중심으로 광고 운영
- 핵심 키워드 순위 방어
- 1위와 2~3위권의 효율 비교를 통한 최적 입찰
- 상품명, 광고문안, 키워드 구조 개선
- 시즌 캠페인 준비 시점 자동 감지
- 반복적인 광고 관리 업무량 감소

### 제품 목표

- AI가 보고서를 쓰는 수준을 넘어 실제 실행 가능한 업무카드를 생성한다.
- 자동 실행은 사람이 정한 규칙 안에서만 수행한다.
- 승인/반려/자동처리 이력을 저장해 다음 룰 개선에 반영한다.
- 처음에는 커피프린트/스티커씨 전용으로 만들고, 나중에 범용 구조로 추상화한다.

## 5. MVP 범위

### 1차 포함

- 네이버 검색광고 API 연동
- 검색광고 캠페인, 광고그룹, 키워드, 입찰가, 성과 데이터 수집
- 최근 90일 성과 분석
- 설날/추석 등 시즌 상품은 전년 동기간 비교
- 키워드별 목표 순위 룰 생성
- 1위 vs 2~3위권 효율 비교
- 브랜드/핵심/시즌/일반/저품질 키워드 분류
- 저위험 입찰 조정 자동 실행 구조
- 승인 필요한 입찰/키워드/문안/상품명 제안 생성
- AI 캐릭터 기반 운영실 화면
- 승인실, 자동처리 이력, 위험 알림

### 1차에서 구조만 만들고 제한적으로 활성화

- 제외 키워드 자동 추가
- 신규 키워드 소액 테스트
- 광고문안 A/B 테스트 등록
- 상품명 변경 제안
- 원가/마진 계산식 입력

### 2차 이후

- 스티커씨 스마트스토어 API 상품/주문/정산 연동
- 커피프린트 영카트 DB 동기화
- 커피프린트 스마트스토어 등록 후 커머스API 연동
- 상품/옵션/수량/인쇄방식별 마진 계산
- 광고 후 공헌이익 기반 자동화
- 블로그 SEO, CRM, 카카오 메시지, 메타/인스타, 구글 광고 확장

## 6. 데이터 소스

### 네이버 검색광고 API

1차 최우선 연동 대상이다.

수집 대상:

- 광고계정
- 캠페인
- 광고그룹
- 키워드
- 현재 입찰가
- 노출
- 클릭
- 비용
- CTR
- CPC
- 전환
- 전환 매출
- 평균 광고 순위 또는 순위 계산 가능한 지표
- 쇼핑검색 키워드 성과
- 예상 입찰가 계열 데이터

사용 목적:

- 핵심 키워드 후보 추출
- 키워드별 성과 등급화
- 1위 방어 후보 선정
- 1위와 2~3위권 효율 비교
- 자동 입찰 조정
- 제외 키워드 후보 발굴
- 신규 키워드 테스트 후보 발굴

### 네이버 커머스API

2차 연동 대상이다.

대상:

- 스티커씨 스마트스토어
- 커피프린트 스마트스토어 등록 후 추가

수집 대상:

- 상품 목록
- 상품명
- 옵션
- 판매가
- 주문
- 상품별 수량
- 정산 예정 금액
- 수수료
- 유입 경로
- 반품/취소 상태

사용 목적:

- 상품별 매출/주문/정산 분석
- 광고 유입과 주문 연결
- 스마트스토어 상품명 최적화
- 마진 계산 보정

### 커피프린트 영카트 DB

2차 또는 3차 연동 대상이다. 직접 운영 DB에 부담을 주지 않도록 **읽기 전용 계정 + 동기화 DB** 방식을 권장한다.

수집 대상:

- 상품
- 옵션
- 주문
- 매출
- 회원
- 유입 경로
- 장바구니
- 결제 상태
- 취소/환불 상태

사용 목적:

- 커피프린트 자사몰 성과 분석
- 스마트스토어와 자사몰 상품 매핑
- 자사몰 구매 전환 기준 광고 성과 판단
- 고객군/재구매/시즌 구매 패턴 분석

### 수동 입력/업로드

수집 대상:

- 옵션별 원가
- 수량별 원가
- 인쇄 방식별 원가
- 봉투/포장/배송지원비
- 목표 마진율
- 자동 실행 허용 범위
- 상품군별 전략 우선순위

초기 입력 방식:

- 엑셀 업로드
- 화면에서 직접 수정
- 계산식 템플릿 저장

## 7. 시즌 기준

설날과 추석은 양력 날짜가 매년 바뀐다. 따라서 날짜를 코드에 고정하지 않고, `MarketingCalendar` 테이블을 둔다.

권장 방식:

- 한국천문연구원 특일 정보 또는 공공데이터포털 특일 API를 사용
- 설날/추석/대체공휴일을 매년 자동 갱신
- 관리자 화면에서 수동 보정 가능
- 시즌 시작/종료 기준은 상품군별로 설정

예시:

| 시즌 | 기준 |
|---|---|
| 설날/연하장 | 명절 D-120부터 준비, D-90부터 광고 강화, D-30부터 입찰 집중 |
| 추석/명절카드 | 명절 D-120부터 준비, D-90부터 광고 강화, D-30부터 입찰 집중 |
| 크리스마스/연말 | 10월부터 준비, 11~12월 집중 |
| 졸업/입학 | 12~3월 집중 |
| 어린이집/유치원 행사 | 2~5월, 9~12월 집중 |
| 생일 답례품 | 상시, 월별 수요 변동 감시 |
| 웨딩/행사 | 봄/가을 집중 |

분석 기준:

```txt
일반 키워드: 최근 90일
시즌 키워드: 최근 90일 + 전년 동기간
설날/추석 키워드: 명절 날짜 기준 D-120 ~ D+14
전년 비교: 명절의 양력 날짜가 아니라 명절 D-day 기준 상대 기간으로 비교
```

예를 들어 추석이 매년 9월 또는 10월로 달라져도 `추석 D-60`, `추석 D-30`, `추석 D-7`처럼 상대 날짜로 비교한다. 설날도 동일하게 `설날 D-90`, `설날 D-30`, `설날 D-7` 기준으로 비교한다.

## 9. 키워드 순위 룰 엔진

이 MVP의 핵심 기능이다.

### 목적

모든 키워드를 1위로 유지하지 않는다. 키워드별로 가장 이익이 좋은 목표 순위를 찾는다.

룰 유형:

```txt
BRAND_DEFENSE: 브랜드 키워드 1위 방어
TOP_1_DEFENSE: 핵심 키워드 1위 방어
TOP_2_TO_3_OPTIMIZE: 2~3위권 이익 최적화
SEASONAL_TOP_TEST: 시즌 기간 1위 테스트
PROFIT_BASED_BID: 공헌이익 기준 입찰
LOW_BID_KEEP: 낮은 입찰 유지
NEGATIVE_CANDIDATE: 제외 후보
PAUSE_CANDIDATE: 중단 후보
DISCOVERY_TEST: 신규 키워드 소액 테스트
```

### 초기 룰 생성 기준

```txt
1. 최근 90일 키워드 성과 수집
2. 시즌 상품은 전년 동기간 성과 추가 비교
3. 설날/추석은 명절 D-day 기준 상대 기간으로 비교
4. 노출, 클릭, 비용, CPC, 전환, 매출, 평균 순위 분석
5. 브랜드명/상품명/시즌어/정보성어/저품질어 분류
6. 1위권과 2~3위권 성과 비교
7. 키워드별 목표 순위와 자동화 한도 생성
```

### 1위 vs 2~3위 효율 비교

비교 항목:

- 평균 순위
- 평균 CPC
- 클릭수
- 전환수
- 전환율
- 매출
- 광고비
- ROAS
- 추정 공헌이익
- 광고 후 마진율

판단 예시:

```txt
'기업 초대장'은 1위일 때 매출은 높지만 CPC가 급등해 공헌이익이 낮다.
2~3위권에서는 클릭수는 줄지만 광고비가 크게 감소하고 공헌이익이 높다.
따라서 목표 순위는 1위 방어가 아니라 2~3위 유지가 적합하다.
```

### 룰 개선

운영 중 룰을 자동으로 바꾸지 않는다. AI가 개선안을 만들고, 자동 허용 범위면 실행하거나 승인실로 보낸다.

룰 개선 트리거:

- 최근 7일/14일/30일 성과가 기존 룰과 다름
- CPC 급등
- 전환율 하락
- 광고 후 마진율 하락
- 시즌 시작/종료
- 경쟁 강도 변화
- 월 광고비 상한 접근
- 상품 원가 또는 판매가 변경

## 10. 마진 계산

### 상품 마진

광고비 제외 기준이다.

```txt
상품 마진 = 판매가 - 제작원가 - 포장비 - 결제수수료 - 플랫폼수수료 - 배송지원비
상품 마진율 = 상품 마진 / 판매가
```

### 광고 후 공헌이익

마케팅 판단 기준이다.

```txt
공헌이익 = 매출 - 상품원가성 비용 - 광고비
광고 후 마진율 = 공헌이익 / 매출
```

### 허용 CPC

입찰 자동화의 기준이다.

```txt
허용 광고비 = 매출 - 원가 - 목표이익
허용 CPA = 주문 1건당 허용 광고비
허용 CPC = 허용 CPA × 전환율
```

초기에는 검색광고 성과만으로 추정하고, 스마트스토어/영카트/원가 계산식이 연결되면 실제 공헌이익 기반으로 전환한다.

## 12. 자동 실행과 승인 정책

### 자동 실행 원칙

AI가 임의로 실행하지 않는다. `AutomationRule` 안에서만 자동 실행한다.

자동 실행 가능 후보:

- 리포트 생성
- 문제 감지
- 키워드 후보 정리
- 광고문안 초안 생성
- 상품명 개선안 생성
- 제한 범위 내 입찰가 조정
- 명확한 저품질 제외 키워드 추가

### 1차 자동 실행 권장 범위

| 작업 | 자동 조건 |
|---|---|
| 입찰가 하향 | 5% 이내, 목표 성과 미달, 최소 데이터 충족 |
| 입찰가 상향 | 5% 이내, 목표 순위 이탈, 최대 CPC/예산/마진 기준 충족 |
| 제외 키워드 후보 등록 | 자동 등록은 보류, 후보 생성만 자동 |
| 광고문안 생성 | 초안 생성만 자동 |
| 상품명 개선 | 초안 생성만 자동 |
| 리포트 생성 | 항상 자동 |

### 승인 필요

- 5% 초과 입찰 변경
- 최대 CPC 초과 변경
- 월 광고비 상한에 가까운 변경
- 예산 증액
- 캠페인 중단/재개
- 상품명 실제 변경
- 광고문안 실제 교체
- 신규 키워드 대량 추가
- 제외 키워드 실제 추가
- 자동화 룰 변경

### 안전장치

- 하루 자동 변경 횟수 제한
- 키워드별 자동 조정 횟수 제한
- 월 광고비 상한
- 최대 CPC
- 목표 마진율 하한
- 전환 없는 키워드 자동 상향 금지
- 핵심 키워드 하향/중단 전 승인 필요
- 자동 실행 이력 저장
- 되돌리기 정보 저장
- 긴급 정지 버튼

## 17. 완료 기준

### MVP 완료 조건

1. 네이버 검색광고 API로 캠페인/광고그룹/키워드 데이터를 가져온다.
2. 최근 90일 키워드 성과를 저장한다.
3. 설날/추석 등 시즌 키워드는 전년 동기간 비교 구조를 가진다.
4. AI가 1위 방어 후보 키워드를 뽑는다.
5. AI가 1위와 2~3위권 효율을 비교한다.
6. 키워드별 목표 순위 룰을 생성한다.
7. 저위험 입찰 조정은 자동 실행 후보로 분류된다.
8. 위험하거나 큰 변경은 승인실에 올라온다.
9. 승인/반려/자동실행 이력이 저장된다.
10. 캐릭터형 운영실에서 AI별 보고를 확인할 수 있다.

### 안전 완료 조건

1. 월 광고비 상한을 초과하는 자동 실행은 불가능하다.
2. 최대 CPC를 초과하는 자동 실행은 불가능하다.
3. 하루 자동 변경 횟수 제한이 적용된다.
4. 핵심 키워드 중단/하향은 승인 없이는 실행되지 않는다.
5. 모든 실행은 이전값과 이후값을 저장한다.
6. API 실패 시 재시도/실패 기록이 남는다.
