# 업무카드형 데스크와 키워드 위임 플랜

## 목표

대표가 긴 결재 문서를 먼저 읽지 않아도, 캐릭터별 데스크에서 담당 업무카드를 빠르게 확인하고 중요한 건만 결재할 수 있게 한다. 검색광고는 상위 안건 하나로 묶지 않고, 실제 판단 단위인 키워드/검색어별 카드로 나눠 보여준다.

## 핵심 결정

1. 검색광고 저성과 안건은 `상위 안건`과 `키워드별 업무카드`를 분리한다.
2. 카드 생성 대상은 전체 키워드가 아니라 조치 판단이 필요한 키워드만이다.
3. 저성과 키워드도 무조건 중지하지 않고 `유지`, `관찰`, `입찰 하향`, `기기 조정`, `시간대 조정`, `추적 확인` 중 하나로 제안한다.
4. 처음 등장한 조치 유형은 대표 첫 승인이 필요하다.
5. 대표가 승인하면서 비슷한 조건은 모아에게 위임할 수 있고, 이후 모아가 처리하더라도 대표에게 계속 보고한다.
6. 인사과에는 캐릭터 사고방식과 보고 형식을 저장하고, 실제 자동 처리 가능 여부는 별도 규칙 엔진이 최종 판단한다.
7. 실제 외부 광고 반영은 이번 플랜에 포함하지 않는다. 화면, 판단 카드, 위임 후보 표시까지만 진행한다.

## 카드 기본 필드

- 키워드 또는 검색어
- 브랜드
- 광고 유형
- 캠페인/광고그룹
- 기기/시간대
- 최근 성과
- 진단
- 제안 조치
- 유지 또는 조정 사유
- 담당 캐릭터
- 처리 경로
- 위임 가능성
- 근거 ID

## 화면 구성

- `/operations`: 오늘 확인해야 할 키워드 업무카드를 최대 6개만 간결하게 노출한다.
- `/characters/gro`: 그로가 만든 검색광고/쇼핑검색광고 키워드 카드를 중심으로 보여준다.
- `/characters/day`: 전환 추적, 근거 누락, 데이터 확인 카드만 보여준다.
- `/characters/moa`: 대표 첫 승인 필요 카드와 위임 후보 카드를 모아 분류한다.
- `/people`: 모아/그로/데이의 기본 지침에 첫 승인, 위임, 계속 보고 원칙을 반영한다.

## 검증 기준

- 키워드 카드에는 실제 키워드 또는 검색어가 반드시 보인다.
- 클릭/비용/주문 또는 직접 전환율 같은 핵심 성과가 1줄 이상 보인다.
- 저성과라도 유지해야 할 수 있다는 사유가 카드에 보인다.
- 대표 첫 승인과 모아 위임 후보가 구분된다.
- 카드 UI는 모바일에서 넘치지 않는다.
- 실제 provider write 경로는 추가하지 않는다.
