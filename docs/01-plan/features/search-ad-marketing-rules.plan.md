# 검색광고 마케팅 룰 보강 플랜

## 목표

네이버 검색광고 보고서 백필 데이터가 모두 쌓인 상태에서, LLM 연결 전에도 대표가 바로 이해하고 조치할 수 있는 마케팅 운영 후보를 만든다. 숫자 판단은 규칙 엔진이 먼저 맡고, LLM은 이후 설명, 예외 판단, 새로운 아이디어 생성에 붙인다.

## 원칙

1. 커피프린트와 스티커씨는 서로 비교하지 않고 각각의 브랜드 안에서 판단한다.
2. 파워링크 검색어와 쇼핑검색광고 검색어는 섞지 않는다.
3. 보고서 종류가 나뉘어 있어도 같은 운영 단위의 비용/클릭/전환/전환매출은 가능한 범위에서 합산한다.
4. 전환매출 누락처럼 데이터 신뢰도 문제가 있으면 광고 성과 판단보다 먼저 점검 카드로 올린다.
5. 카드의 첫 문장은 숫자 진단, 두 번째 정보는 실제 조치 후보가 되게 한다.
6. 공식 도움말 기준으로 전환 보고서가 아닌 상세 보고서는 보조 근거로만 사용한다. 전환/ROAS/저효율 자동 판단은 전환 근거가 확인되는 보고서에 한정한다.

## 룰 상세

### 데이터 신뢰도

- 조건: 전환수 `> 0`, 전환매출 `= 0`, 비용이 브랜드 기준 최소 비용 이상.
- 결과: `점검 필요`.
- 이유: 매출이 실제로 없는 것이 아니라 전환매출 전달이 누락된 상태일 수 있다.
- 첫 조치: `전환 점검 후보`로 분리해 전환 코드, 전환 유형, 주문 금액 전달 상태를 먼저 확인한다.

### 파워링크 검색어

- 저효율 검색어: 제외어 후보 또는 입찰 하향 후보.
- 우수 검색어: 등록 키워드 추가, 유사 키워드 확장 후보.
- 주의: 방어 키워드, 브랜드 키워드, 시즌 준비 키워드는 바로 제외하지 않는다.

### 쇼핑검색광고 검색어와 소재

- 저효율 검색어: 상품명, 대표 이미지, 랜딩 상품 적합도 점검 후보.
- 저효율 광고 소재: 광고 문구와 랜딩 URL 적합도 점검 후보.
- 쇼핑검색광고 확장소재: 현재 `ADEXTENSION` 수집값은 노출 중심으로 보며, `ADEXTENSION_CONVERSION` 연결 전까지 클릭 없음/저효율 후보로 올리지 않는다.
- 우수 검색어: 상품 확장, 예산 확대, 유사 상품 연결 후보.
- 주의: 검색어 상세 보고서와 전환 상세 보고서를 합산해 봐야 한다.

### 보고서 신뢰도 기준

- 공식 근거: [네이버 대용량 보고서 도움말](https://ads.naver.com/help/faq/367), [네이버 Search AD API StatReport 샘플](https://github.com/naver/searchad-apidoc/blob/master/java-sample/src/main/java/com/naver/searchad/api/rest/StatReports.java), [쇼핑검색 검색어/전환 상세 릴리스 노트](https://naver.github.io/searchad-apidoc/release/2018/02/21/release-note/).
- 네이버 도움말 기준으로 `광고성과 보고서`는 광고 키워드와 소재의 성과를 제공하고, `광고성과 상세 보고서`는 지역/시간대별로 더 잘게 쪼갠 보조 성과를 제공한다.
- `전환 보고서`, `전환 상세 보고서`, `쇼핑검색 검색어 전환 상세 보고서`, `타게팅 전환 보고서`는 프리미엄 로그 분석 전환 근거가 있어야 제공된다.
- 따라서 `광고성과 상세 보고서(AD_DETAIL)`만 보고 "전환 없음" 또는 "ROAS 0%"으로 자동 판단하지 않는다.
- 같은 광고라도 기기/매체별 성과가 다르면 별도 세부 분해로 보여준다. 대표는 비용 비중이 큰 기기/매체부터 입찰, 예산, 노출 범위를 조정한다.
- `AD_DETAIL`은 원인 추적 화면에서 보조 근거로 보여주되, 실제 후보 생성은 `AD`, 검색어 성과/전환 쌍, 타게팅 성과/전환 쌍처럼 판단 가능한 보고서군만 사용한다.

### 기기/시간대/타게팅

- 성과 보고서와 전환 보고서를 같은 타게팅 ID 기준으로 합산한다.
- `P/M` 기기는 `PC/모바일`로 표시한다.
- `GNF`, `GNM`, `AG3539`, `SDMON0820` 같은 코드는 여성, 남성, 35~39세, 월요일 시간대처럼 보여준다.

### 키워드 효율 분석

- 목적: 같은 키워드 또는 검색어 안에서 효율이 좋은 `기기 + 매체 + 시간대 + 지역` 조합을 찾고, 낮은 조합은 축소 후보로 분리한다.
- 공식 근거: 네이버 광고관리 기본 보고서는 `PC/모바일`, `요일`, `시간대`, `지역` 상세 데이터를 제공하고, 대용량 보고서는 ID와 지표를 조합해 자유롭게 합산할 수 있다.
- 참고 문서: `https://ads.naver.com/help/faq/265`, `https://ads.naver.com/help/faq/1239`.
- 판단 단위: `브랜드 + 광고유형 + 캠페인 + 광고그룹 + 키워드/검색어/광고 소재 + 기기 + 매체 + 시간대 + 지역`.
- 쇼핑검색 검색어는 `SHOPPINGKEYWORD_DETAIL`과 `SHOPPINGKEYWORD_CONVERSION_DETAIL`을 함께 본다.
- 파워링크 등록 키워드는 `AD_DETAIL`과 `AD_CONVERSION_DETAIL`을 함께 본다.
- 신뢰도 높음: 클릭 20회 이상 또는 비용 2만원 이상이고 전환 3건 이상.
- 신뢰도 중간: 클릭 10회 이상 또는 비용 1만원 이상. 전환 1건만으로는 확장하지 않고 관찰한다.
- 신뢰도 낮음: 위 조건 미달. 자동 확장하지 않고 관찰한다.
- 확장 검토: ROAS 350% 이상, 신뢰도 높음.
- 유지: ROAS 250% 이상, 신뢰도 중간 이상.
- 축소 검토: 전환 0이고 클릭 20회 이상 또는 비용 2만원 이상, 또는 비용 2만원 이상이며 ROAS 180% 미만.
- `/keyword-insights` 화면에서 `가장 효율 좋은 조합`, `낭비 가능성이 큰 조합`, `관찰할 조합`을 분리해 보여준다.
- 자동 실행은 하지 않고, 이후 입찰/시간대/매체 조정 미리보기와 대표 승인 플로우로 연결한다.

### 시즌/행사

- 검색어, 키워드, 광고그룹, 캠페인명에서 시즌/행사 단서를 찾는다.
- 1차는 카드 표시만 한다.
- 다음 단계에서는 음력 명절 윈도우, 전년도 같은 시즌, 데이터랩 상승 흐름과 연결한다.

### 중복 키워드 처리

- `/keywords`에서 같은 브랜드/광고유형 안의 중복 키워드 후보를 묶어 보여준다.
- 후보별 실행은 바로 삭제하지 않고 `끄기 미리보기`를 먼저 만든다.
- 대표가 승인 버튼을 누르면 기존 write gate를 확인한 뒤 네이버 키워드 `userLock` 변경 또는 차단 이력을 남긴다.
- 모든 결과는 `/action-logs`에 남겨 어떤 키워드를 왜 껐는지 추적한다.
- 실제 키워드 삭제, 제외어 등록, 키워드 신규 추가는 별도 승인 플로우로 분리한다.
- 최종 실행 전에는 한 번 더 확인 단계를 거쳐 실수 클릭을 막는다.

## 다음 작업 후보

1. 실제 키워드 삭제는 `끄기 후 관찰 기간`을 둔 뒤 별도 확인으로 연다.
2. 마진율과 재고 상태를 연결해 “매출”이 아니라 “이익” 기준 룰을 추가한다.
3. LLM이 읽을 evidence packet을 카드별로 압축해 설명/예외/아이디어를 생성한다.
