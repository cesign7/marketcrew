# product-image-studio Planning Document

> **Summary**: 기존 검색광고 운영 기능과 분리된 `상품 이미지 스튜디오`를 만든다. 1차 MVP는 카드 + 봉투 + 봉합스티커 인쇄물 세트를 대상으로, 업로드한 디자인을 보존하면서 상품 목록과 상세페이지에 쓸 설정샷을 생성, 비율 변경, 다운로드하는 화면이다.

## Context Anchor

| 항목 | 내용 |
|------|------|
| WHY | 인쇄물 상품은 디자인 정확도가 중요하지만 상품 목록, 대표이미지, 상세페이지에는 세트가 놓인 자연스러운 설정샷이 필요하다. |
| WHO | 카드, 초대장, 연하장, 스티커, 봉투 같은 인쇄물 상품을 판매하는 대표 또는 운영자. |
| RISK | 실제 생성 provider 비용, 업로드 디자인 훼손, 타사 이미지 복제 오해, 스마트스토어/AIPRESSO 같은 외부 원장 쓰기. |
| SUCCESS | `/product-image-studio`에서 카드 형식 선택, 디자인 업로드, 콘셉트 추천, 생성/fake 생성, 결과 확인, 비율 변경, 개별/ZIP 다운로드가 가능하다. |

## 1. MVP Scope

### In Scope

- `/product-image-studio` 별도 화면과 `product-image-studio` bounded context.
- 상품군은 카드 + 봉투 + 봉합스티커 세트로 고정한다.
- 카드 형식은 `접이식 카드`와 `엽서형 카드`를 모두 지원한다.
- 접이식 카드는 닫힌 컷, 펼친 컷, 반쯤 열린 컷, 세운 컷 같은 자세를 고를 수 있다.
- 엽서형 카드는 접힘이나 안쪽 면을 억지로 만들지 않고 평면/기울임/벽 기대기 중심으로 다룬다.
- 필수 업로드는 카드, 봉투, 봉합스티커 디자인 이미지다.
- 선택 업로드는 접이식 카드 안쪽, 뒷면, 봉투 플랩, 참고 분위기 이미지다.
- 출력은 `세트컷`, `카드 단독컷`, `봉투 단독컷`, `봉합스티커 단독컷` 네 가지다.
- 결과는 설정샷 그룹으로 보여주고, 유사 버전 생성 결과도 같은 그룹에서 비교한다.
- 지원 비율은 1:1, 4:5, 3:4, 16:9, 사용자 지정 크기다.
- 다운로드는 개별 이미지와 ZIP 다운로드를 제공한다.
- 기본 상태의 provider gate는 닫혀 있으며 실제 provider 호출 차단을 기본값으로 둔다.

### Out of Scope

- 스마트스토어 상품 조회/수정 자동 연결.
- AIPRESSO 디자인 자동 불러오기.
- 자사몰, 인스타그램, 블로그 자동 발행.
- 배너 생성.
- 실제 provider 실생성 기본 활성화.
- 외부 원장 쓰기 금지: 스마트스토어, AIPRESSO, 영카트, 네이버 광고, 데이터랩에는 쓰지 않는다.
- 타사 참고 이미지를 그대로 복제하거나 라이선스 판단을 자동으로 대신하는 기능.

## 2. Product Contract

| 출력 | 목적 | 보존 기준 |
|------|------|-----------|
| 세트컷 | 카드, 봉투, 봉합스티커가 함께 놓인 대표/상세 이미지 | 각 상품 비율과 디자인을 유지하고 세트 구성만 자연스럽게 배치한다. |
| 카드 단독컷 | 접이식 카드 또는 엽서형 카드의 단독 설정샷 | 접이식은 실제 접힌 형태를 허용하고, 엽서형은 접힘을 만들지 않는다. |
| 봉투 단독컷 | 봉투 상품의 단독 설정샷 | 봉투 앞면과 플랩 방향을 훼손하지 않는다. |
| 봉합스티커 단독컷 | 봉합스티커 또는 봉투에 붙은 설정샷 | 원형/사각형 스티커 형태를 유지하고 붙은 장면은 자연스러운 원근을 적용한다. |

## 3. Architecture Guardrails

- Search Ad 도메인 타입과 화면 필터를 가져오지 않는다.
- 업로드 파일은 서버 API에서만 처리하고 브라우저에는 안전한 preview URL만 돌려준다.
- provider gate가 닫혀 있으면 생성 API는 423 차단 응답을 돌려주고 실제 provider 호출을 하지 않는다.
- fake provider는 로컬 QA와 자동 테스트에서만 사용한다.
- provider 상태 화면은 키, 모델 값, env 값을 노출하지 않고 설정 여부만 보여준다.
- 비율 변경 생성은 기존 결과를 수정하지 않고 새 결과 레코드를 만든다.
- 운영 스모크는 로그인 보호와 상태 API 보호만 확인하고 생성 비용이 드는 엔드포인트를 호출하지 않는다.

## 4. Future Integration

- 스마트스토어: 상품 URL/상품 ID에서 원본 이미지와 설명을 읽어 프로젝트 초안을 만드는 read-only adapter를 붙인다.
- AIPRESSO: 템플릿 디자인과 인쇄물 슬롯을 자동으로 불러와 업로드 단계를 줄인다.
- 자사몰: 결과 이미지와 상품 설명 배치를 상세페이지 초안으로 저장한다.
- SNS/블로그: 같은 결과를 인스타그램, 블로그 콘텐츠 비율과 문구로 확장한다.
- 실제 provider: 사용자 승인과 비용 안내, 낮은 비용의 live smoke, 실패 재시도 정책이 준비된 뒤 연다.

## 5. Verification

- 도메인 계약, 파일 저장, provider gate, 프로젝트/업로드/콘셉트/생성/다운로드 API 테스트.
- UI 테스트: 위저드, 생성 패널, 결과 갤러리, 다운로드 패널, 상태 패널.
- 브라우저 QA: 로컬 fake provider로 업로드부터 ZIP 다운로드까지 데스크톱과 모바일에서 확인.
- 운영 안전 스모크: `/product-image-studio` 로그인 보호와 `/api/product-image-studio/provider-status` 401 보호 확인.
